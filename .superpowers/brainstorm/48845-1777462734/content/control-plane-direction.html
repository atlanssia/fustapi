Design commitments

  • Work surface first: no hero, no marketing copy.
  • Light neutral UI for scanability, with restrained blue actions.
  • Providers and routes are real CRUD workflows with clear empty/error/loading states.
  • Health only reports gateway status unless new provider health APIs exist.

Risks contained

  • API keys are never rendered in full, even though the backend returns them.
  • Route provider order is shown as priority order.
  • Mobile falls back to stacked records instead of squeezed tables.
  • Every destructive action requires confirmation.
